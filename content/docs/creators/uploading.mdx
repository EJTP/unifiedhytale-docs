This guide walks you through uploading your first project to UnifiedHytale.

## Before You Start

Make sure you have:

- Creator status (see [Becoming a Creator](/docs/creators/becoming-creator))
- Project files ready (`.zip`, `.jar`, `.rar`, or `.7z`)
- Icon image (256x256px recommended)
- Banner image (1920x1080px recommended)

## Step-by-Step Upload

### 1. Start New Project

From your [Dashboard](https://unifiedhytale.com/dashboard), click **"New Project"**.

### 2. Basic Information

| Field | Description |
|-------|-------------|
| **Type** | Plugin, Data Pack, Asset Pack, World, Prefab, or Modpack |
| **Name** | 3-100 characters, must be unique |
| **Slug** | URL-friendly name (auto-generated) |
| **Summary** | 10-150 characters, shown on cards |

### 3. Description

Write a detailed description using [Markdown](/docs/creators/markdown):

- What does your project do?
- How to install and use it
- Known issues or limitations
- Credits and acknowledgments

<Callout type="info">
Minimum 100 characters required. More detail = more downloads!
</Callout>

### 4. Media

**Icon** (required)
- Square image, 256x256px recommended
- Max 5MB, PNG/JPG/WebP
- Shows in search results and cards

**Banner** (recommended)
- Wide image, 1920x1080px recommended
- Max 5MB, PNG/JPG/WebP
- Shows on project page header

**Gallery** (optional)
- Up to 10 additional images
- Screenshots, feature highlights
- Max 5MB each

### 5. Categories

Select 1-3 categories that best describe your project:

- Gameplay, Aesthetic, Technical
- Adventure, Building, Utility
- And more...

### 6. Upload Files

Upload your project file:

- Supported formats: `.zip`, `.jar`, `.rar`, `.7z`
- Size limits vary by type (see [Project Types](/docs/creators/project-types))
- Include README if possible

### 7. Version Details

For your initial version:

- **Version Number** - Use semantic versioning (e.g., 1.0.0)
- **Changelog** - What's in this release
- **Game Versions** - Compatible Hytale versions
- **Dependencies** - Other required projects

### 8. Review & Submit

- Preview how your project will look
- Check all information is correct
- Click **"Submit for Review"**

## After Submission

Your project enters the moderation queue:

1. Automated virus scan
2. Staff review (24-48 hours)
3. Approval or feedback email

<Callout type="info">
You'll receive email notifications at each step of the review process.
</Callout>

---
title: Project Requirements
description: Technical requirements for uploading projects to UnifiedHytale.
---

Make sure your projects meet these requirements before uploading.

## Text Requirements

| Field | Min | Max | Notes |
|-------|-----|-----|-------|
| Name | 3 | 100 | Must be unique |
| Summary | 10 | 150 | Shown on cards |
| Description | 100 | 50,000 | Markdown supported |

## Image Requirements

### Icon
- **Size:** 256x256px recommended
- **Format:** PNG, JPG, WebP
- **Max file size:** 5MB
- **Aspect ratio:** Square (1:1)

### Banner
- **Size:** 1920x1080px recommended
- **Format:** PNG, JPG, WebP
- **Max file size:** 5MB
- **Aspect ratio:** 16:9

### Gallery Images
- **Count:** Up to 10 images
- **Format:** PNG, JPG, WebP
- **Max file size:** 5MB each

<Callout type="info">
High-quality images significantly increase downloads. Invest time in good visuals!
</Callout>

## File Requirements

### Size Limits

| Project Type | Max Size |
|--------------|----------|
| Plugin | 200MB |
| Data Pack | 50MB |
| Asset Pack | 100MB |
| World | 500MB |
| Prefab | 50MB |
| Modpack | 500MB |

### Formats

Accepted archive formats:
- `.zip` (recommended)
- `.jar` (for plugins)
- `.rar`
- `.7z`

### File Structure

Your archive should include:
- Main content files
- README or documentation
- License file (if applicable)
- No unnecessary files (temp files, .DS_Store, etc.)

## Content Guidelines

All projects must:

1. **Be original** or properly licensed
2. **Not contain malware** or malicious code
3. **Match the description** accurately
4. **Work as advertised** (no broken downloads)
5. **Follow community guidelines**

## Metadata Requirements

### Categories
Select 1-3 relevant categories that describe your project.

### Tags
Add up to 10 tags to help users find your project.

### Dependencies
List all required dependencies accurately.

### Game Versions
Specify which Hytale versions are compatible.
